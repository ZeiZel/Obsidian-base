# UI Frameworks

Экосистема UI-фреймворков для React значительно выросла за последние годы. Каждый фреймворк занимает свою нишу - от полноценных дизайн-систем с сотнями компонентов до минималистичных headless-решений, дающих полный контроль над стилями.

## Сравнение фреймворков

| Фреймворк | Компоненты | Bundle size | Кастомизация | TypeScript | Дизайн-система | Фреймворки |
|---|---|---|---|---|---|---|
| Ant Design | 60+ | ~350 KB | Design Tokens | Нативный | Ant Design | React |
| MUI | 50+ | ~300 KB | Theme + sx | Нативный | Material Design | React |
| Mantine | 100+ | ~200 KB | CSS Variables | Нативный | Собственная | React |
| shadcn/ui | 40+ | ~0 KB (copy) | Tailwind + CSS Vars | Нативный | Radix-based | React, Next.js |
| Chakra UI | 50+ | ~160 KB | Theme tokens | Нативный | Собственная | React |
| Base UI | 25+ | ~50 KB | Любой CSS | Нативный | Headless | React |

> [!info] Bundle size
> Указан примерный размер при полном импорте. На практике tree-shaking значительно уменьшает итоговый бандл. shadcn/ui не является npm-пакетом - компоненты копируются в проект, поэтому размер зависит от того, сколько компонентов добавлено.

## Навигация

- [[Ant Design]] - enterprise-решение от Alibaba с мощными таблицами и формами
- [[MUI]] - реализация Material Design от Google с обширной экосистемой
- [[Mantine]] - современный фреймворк с PostCSS, богатым набором хуков и модулей
- [[Shadcn UI]] - copy-paste компоненты на базе Radix UI и Tailwind CSS
- [[Chakra UI]] - доступные composable-компоненты с простым API
- [[Base UI]] - headless-компоненты от команды MUI для полного контроля над стилями

## Когда какой фреймворк использовать

Enterprise-приложения с тяжелыми таблицами и формами - Ant Design или MUI. Оба предлагают DataGrid, DatePicker и другие сложные компоненты из коробки. Ant Design традиционно сильнее в табличных интерфейсах, MUI лучше интегрируется с Material Design.

Быстрый старт с хорошими defaults - Mantine. Фреймворк покрывает максимум сценариев из коробки: формы, уведомления, rich text editor, датапикеры. При этом не использует CSS-in-JS runtime, что положительно сказывается на производительности.

Полный контроль над стилями и кодом - shadcn/ui. Компоненты копируются в проект и становятся частью кодовой базы. Идеально для команд с собственной дизайн-системой, которым нужна accessible-основа без vendor lock-in.

Максимальная простота и доступность - Chakra UI. Минимальный boilerplate, интуитивный API со стилями через пропсы, встроенная поддержка темной темы.

Headless-подход с любой CSS-библиотекой - Base UI. Предоставляет логику и доступность без стилей. Подходит для проектов с уникальным дизайном, где готовые стили только мешают.

> [!summary] Общая рекомендация
> Для нового проекта стоит начать с shadcn/ui или Mantine. Первый дает максимальную гибкость и контроль, второй - максимальную функциональность из коробки. Для enterprise с большими таблицами и формами лучше смотреть на Ant Design или MUI с их DataGrid-компонентами.
