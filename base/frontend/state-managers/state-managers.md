## Обзор

Управление состоянием - одна из центральных задач при разработке фронтенд-приложений. С ростом сложности приложения встроенных средств фреймворка часто недостаточно, и на помощь приходят специализированные библиотеки.

Основные подходы к управлению состоянием:
- Flux / Redux-like - однонаправленный поток данных, единый стор, экшены и редьюсеры
- Атомарный - состояние разбивается на мелкие атомы, которые компонуются в граф зависимостей
- Proxy-based - мутабельный API с автоматическим отслеживанием изменений через Proxy
- Событийный - состояние управляется через явные связи между событиями, сторами и эффектами
- Signals - тонкогранулярная реактивность с автоматическим отслеживанием зависимостей

---

## Содержание

- [[Zustand]] - минималистичный стор на хуках
- [[MobX]] - реактивное управление через наблюдаемые объекты
- [[Jotai]] - атомарный подход, вдохновлённый Recoil
- [[Effector]] - событийная архитектура с декларативными связями
- [[Valtio]] - proxy-based мутабельный API с иммутабельными снапшотами
- [[Signals]] - паттерн тонкогранулярной реактивности

## Когда что использовать

### Zustand

Подходит для большинства проектов. Минимальный размер, простой API, не требует провайдеров. Хорош для команд, которым нужен понятный flux-подход без бойлерплейта Redux.

### MobX

Выбор для проектов с классовой архитектурой и сложными доменными моделями. Автоматическое отслеживание зависимостей снижает количество ручной оптимизации ре-рендеров.

### Jotai

Оптимален, когда состояние естественно разбивается на независимые атомы. Интеграция с React Suspense делает его удобным для приложений с асинхронными данными. Хорош для замены множества `useState` + `useContext`.

### Effector

Предпочтителен для крупных проектов с SSR и сложной бизнес-логикой. Декларативные связи через `sample` и `split` дают прозрачный граф зависимостей. Требует времени на освоение, но окупается на масштабе.

### Valtio

Удобен, когда состояние содержит глубоко вложенные объекты. Мутабельный API снижает количество кода при обновлении вложенных структур. Подходит для проектов, где разработчики привыкли к мутабельному стилю.

### Signals

Паттерн тонкогранулярной реактивности, встроенный в Solid.js и Angular 17+. Для React доступен через Preact Signals. Оптимален для приложений с высокими требованиями к производительности обновлений.

> [!summary] Общее правило
> Для нового React-проекта среднего размера начинайте с Zustand. Переходите на Jotai, если состояние атомарно по природе. Рассмотрите Effector для крупных проектов с SSR. Используйте Valtio, если предпочитаете мутабельный стиль. MobX актуален для legacy-проектов или классовой архитектуры.
